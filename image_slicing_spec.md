# 이미지 슬라이서 작업지시서 (Image Slicer Specification)

본 문서는 세로로 긴 상세페이지 이미지를 논리적인 구분선(가로선)을 기준으로 하위 이미지로 쪼개는 프로그램 개발을 위한 지시서입니다.

## 1. 프로젝트 개요
- **목표**: 대용량 세로 이미지(상세페이지)를 웹/모바일 최적화를 위해 여러 개의 슬라이스로 자동 분할.
- **핵심 로직**: 이미지 내의 가로 공백(Gap) 또는 일정 색상의 가로선을 감지하여 절단 지점을 결정함.

## 2. 주요 기능
1. **이미지 로드**: JPG, PNG 등 일반적인 이미지 포맷 지원.
2. **구분선 감지**:
    - 가로 방향의 픽셀들을 분석하여 색상 변화가 거의 없는(단색) 행(Row)을 감지.
    - 연속된 단색 행들을 하나의 '절단 가능 영역'으로 그룹화.
3. **이미지 분할**:
    - 감지된 영역의 중앙을 기준으로 이미지를 수평 절단.
    - 사용자가 지정한 최소 슬라이스 높이(Threshold)를 고려하여 너무 작게 쪼개지지 않도록 조절.
4. **결과 저장**:
    - `image_slicer/img/` 폴더 내에 순차적으로 파일 저장 (예시: `output_01.png`, `output_02.png`).

## 3. 기술 스택
- **언어**: Python 3.x
- **라이브러리**: 
    - `Pillow` (PIL): 기본 이미지 처리 및 저장.
    - `NumPy`: 고속 행렬 연산을 통한 픽셀 데이터 분석.
    - (선택사항) `OpenCV`: 고급 이미지 처리 필요 시.

## 4. 제약 사항 및 고려 사항
- **배경색 처리**: 흰색뿐만 아니라 상세페이지 고유의 배경색(연한 회색, 연한 녹색 등)도 감지할 수 있도록 오차 범위(Variance/Threshold) 설정 기능 필요.
- **슬라이스 크기**: 너무 빈번한 절단을 방지하기 위해 최소 높이(Min Height) 설정값 적용.
- **이미지 품질**: 저장 시 원본의 선명도 유지 (Lossless 또는 고화질 저장).

## 5. 작업 단계
1. `src/` 폴더 내에 핵심 모듈 작성.
2. 분석 단계에서 절단 지점을 시각화하여 확인하는 기능 추가.
3. 최종 배치 처리(Batch processing) 기능 구현.
